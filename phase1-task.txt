## Context
The POC memory pipeline is append-only -- MEMORY.md files grow forever with stale entries dominating retrieval. Your job is to implement the entry schema and compaction system that fixes this.

All files you write go under: poc/projects/POC/ (relative to your CWD, which is the session worktree).

## Files to CREATE:

### 1. poc/projects/POC/scripts/memory_entry.py
Entry schema with YAML frontmatter. Required fields on every entry: id, type, domain, importance, phase, status, reinforcement_count, last_reinforced, created_at, content.

Do NOT use pyyaml. Parse YAML frontmatter with Python's built-in re module only.

Required class:
  MemoryEntry dataclass with fields:
    id: str (uuid4)
    type: str (declarative|procedural|directive|corrective)
    domain: str (task|team)
    importance: float (0.0-1.0)
    phase: str (project phase, e.g. 'specification', 'implementation')
    status: str (active|retired|compacted)
    reinforcement_count: int
    last_reinforced: str (ISO date e.g. '2026-03-01')
    created_at: str (ISO date)
    content: str (the learning text)

Required functions:
- make_entry(content, type='procedural', domain='team', importance=0.5, phase='unknown') -> MemoryEntry
  Creates entry with uuid4 id, today's date for last_reinforced and created_at, status='active', reinforcement_count=0

- parse_frontmatter(text: str) -> tuple[dict, str]
  Parses '---\nkey: val\n---\ncontent' format. Returns (metadata_dict, content_str).
  Types: importance->float, reinforcement_count->int, other fields->str.
  Raises ValueError if no frontmatter found.

- serialize_entry(entry: MemoryEntry) -> str
  Returns YAML frontmatter block + content. Exact format:
    ---
    id: <uuid>
    type: procedural
    domain: team
    importance: 0.7
    phase: specification
    status: active
    reinforcement_count: 0
    last_reinforced: '2026-03-01'
    created_at: '2026-03-01'
    ---
    <content text here>

- parse_entry(text: str) -> MemoryEntry
  Parses one frontmatter+content block. Old-format entries (no frontmatter, e.g. '## [date] Learning') get
  defaults: domain='team', importance=0.5, phase='unknown', status='active', reinforcement_count=0,
  id=new uuid4, dates=today. Missing optional fields in frontmatter also get defaults.

- parse_memory_file(text: str) -> list[MemoryEntry]
  Splits full MEMORY.md text into individual entries on YAML '---' boundaries.
  Handles both old-format (## [date] header style) and new YAML frontmatter entries.
  Returns list of MemoryEntry. Empty input returns [].

- serialize_memory_file(entries: list[MemoryEntry]) -> str
  Joins serialized entries with '\n\n' separator. Returns empty string for empty list.

### 2. poc/projects/POC/scripts/compact_memory.py
Compaction engine. Reads MEMORY.md, deduplicates entries, rewrites atomically.

CLI: compact_memory.py --input <path> [--output <path>]
If --output omitted, overwrites --input in-place.

Algorithm:
1. Parse all entries via parse_memory_file()
2. Deduplicate by id: same id -> keep highest importance version
3. Content-similarity dedup: for ACTIVE entries, compute Jaccard similarity on word token sets.
   If Jaccard > 0.8, merge: keep higher importance, reinforcement_count=max of both,
   last_reinforced=most recent, content=longer string, status='compacted'
4. Exclude retired entries (status == 'retired' are dropped)
5. Write atomically: write to tempfile, then os.replace() to output path
6. Return (input_count, output_count)

Importable: compact_file(input_path: str, output_path: str = None) -> tuple[int, int]

Must be idempotent: two compact_file() calls on same file produce same result both times.
