**Pre-mortem Risk:** Regression testing of existing tests and ops scripts may fail silently (Risk 4).
**Outcome:** Test execution mitigation was not evident in execution log (code review only was performed)
**Learning:** The pre-mortem called for "Run the full test suite (`pytest`) after each principle" and explicit testing of ops scripts. The execution log shows code review + syntax checks, but no actual test suite runs. Project-lead noted "I recommend running those verification steps directly before proceeding"—implying they were skipped in the liaison context. This is a workflow gap: pre-mortems assume test execution, but parallel dispatch contexts may have sandbox restrictions or implicit assumption that sub-agents own their testing.
**Action:** If a pre-mortem mitigation depends on test execution, either (a) make test results a **hard completion requirement** for the dispatch (add to task description: "tests must pass before dispatch is complete"), or (b) add local assertions or self-tests to the code itself so correctness can be validated without external test harness. Account for sandbox/dispatch restrictions in the mitigation design.
