## [2026-03-09] Corrective Learning
**Error Pattern:** Sub-agent dispatch for file edits returned success reports without actually modifying files. Multiple retries with `dispatch.sh` and liaison agents failed to complete surgical edits to `memory_indexer.py` and `summarize_session.py`.

**Recovery:** Abandoned sub-agent dispatch midway and switched to direct agent implementation. All four target files were then successfully modified in a single direct dispatch without verification gaps.

**Root Cause:** Sub-agents hitting tool limits or incomplete task comprehension when receiving multi-edit tasks via dispatch mechanism. File modification delegation doesn't guarantee persistence.

**Prevention:** For focused file write tasks in worktrees, prefer direct agent dispatch over liaison/sub-agent intermediaries. Reserve sub-agent delegation for research/exploration phases only. If delegating file edits, verify actual file contents post-dispatch rather than trusting completion summaries.

## [2026-03-09] Corrective Learning
**Error Pattern:** `dispatch.sh` invocation with task arguments failed across multiple syntaxes:
- `--task "multiline string"` (shell expansion issues)
- `--task-file` with various quoting/redirection approaches
- Escaped variables and subshells

**Recovery:** Eventually abandoned dispatch.sh entirely and used direct agent invocation for file edits.

**Root Cause:** Dispatch script argument parsing incompatible with complex task descriptions; heredoc, variable expansion, and quoting techniques all encountered shell interpretation issues.

**Prevention:** Avoid using `dispatch.sh` for complex multi-line task specifications in worktrees. Use direct agent invocation or write task specifications to files within the worktree directory structure before dispatch, not in temp files outside the working context.
