## [2026-03-10] Corrective Learning

**Error Pattern:** Writing team created documentation with a plausible but unverified git clone URL (`https://github.com/dlewissandy/teaparty.git`). The main agent couldn't verify the actual remote URL directly and discovered this after content was already drafted.

**Recovery:** Rather than blocking, explicitly flagged the URL as a placeholder in the editorial dispatch task, allowing editorial team to correct it during review. The editorial team then updated all three OS sections with the verified URL.

**Root Cause:** Writing team lacked access to the actual repository remote in their execution context, so they invented a reasonable-looking URL. Verification attempt (`git remote get-url origin`) was attempted too late — after content was already written.

**Prevention:** For multi-team dispatch workflows, gather external context (git remotes, URLs, credentials) **before** dispatching content creation. Either: (1) verify and pass the actual remote as a task parameter, or (2) explicitly request in the dispatch that agents verify against the source environment before writing. Avoid letting writing teams invent external URLs.
