## Summary for Next Intent Session

**Key improvements for future state hygiene (or similar phased) intents:**

1. **Define acceptance tests, not just descriptions.** Instead of "Resumability works in practice," write: "Principle 3 complete when: a session resumes from state.json alone, no streams read, and a human can continue work. Test: start session, checkpoint, kill process, resume from state.json, verify phase/dispatch/directory state reconstructed."

2. **Make dependencies explicit.** State which principles block others. Don't start Principle 3 until Principle 4 is done if Principle 4 is a prerequisite for validation.

3. **Verify open questions before proceeding.** Flag: "BLOCKED: awaiting validation of CLAUDE_CODE_TASK_LIST_ID" if the prerequisite is unmet, rather than implementing as if it works.

4. **Call out architectural deviations.** When agents choose a different design (e.g., centralized `record_approval.py` vs. modifying two files), they should flag it: "Architecture choice: we chose X because of Y."

5. **Include output format samples in verification.** Show what `summary.md` actually contains, not just that it exists.

6. **Test regression at the end.** List which existing scripts/tests must pass after all principles are complete. Verify them explicitly.
