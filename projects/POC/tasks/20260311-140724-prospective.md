**Learning:** Pre-mortem predicted subprocess contract violations but expected issues with argument passing. Actual failure: `session.py` calls `memory_indexer.py` with flags that don't exist. This is a semantic error in the call site, not a plumbing issue. The suggested contract test (comparing outputs between bash and Python versions) would have caught this, but only if it ran the full session flow end-to-end—unit-level contract tests wouldn't surface it.
**Action:** For subprocess-heavy refactoring, expand contract tests to full integration level and verify not just argument syntax but semantic correctness (do the flags exist? is the logic for constructing them correct?).
