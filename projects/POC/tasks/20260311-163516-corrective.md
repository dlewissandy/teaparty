## [2026-03-11] Permission Boundaries Misunderstanding
**Error Pattern:** Agent attempted to dispatch execution work to sub-agents via `SendMessage`, expecting them to run `dispatch.sh`, then realized "The sub-agents don't have Bash permissions."

**Recovery:** Switched to running dispatch calls directly from the parent agent using direct `Bash` tool instead of delegating to sub-agents.

**Root Cause:** Misunderstood that sub-agents spawned for specialized work (coding-liaison, etc.) may not have the same tool permissions as the parent orchestrator agent.

**Prevention:** Before delegating tool-dependent work to sub-agents, explicitly verify their permission model. For orchestration tasks requiring Bash, execute directly from parent rather than delegating.

## [2026-03-11] Shared File Path Accessibility Across Sandboxes
**Error Pattern:** Agent wrote task descriptions to `/tmp` and attempted to read them back via `Read` tool, then discovered "/tmp isn't accessible via Read" and couldn't coordinate between parallel Bash calls this way.

**Recovery:** Switched to passing task content inline directly to `dispatch.sh` command-line arguments instead of via temporary files.

**Root Cause:** `/tmp` files may not be accessible from within the execution sandbox or may have cross-process visibility constraints; assumption that `/tmp` would work as inter-process coordination failed.

**Prevention:** For temporary coordination between tool calls, use worktree-local paths or pass content inline. Do not rely on `/tmp` or system temp directories for cross-sandbox communication.

## [2026-03-11] Task String Construction Complexity Causing Confusion
**Error Pattern:** Multiple attempts to build complex task strings using inline Python, heredoc redirection, and shell escaping resulted in truncated or unclear command states, requiring several retry iterations.

**Recovery:** Simplified to direct `dispatch.sh` calls with plain text task descriptions (no special characters, no nested shell constructs).

**Root Cause:** Over-engineering the task dispatch mechanism by mixing multiple layers of escaping (shell metacharacters, Python string literals, argument quoting). This made debugging difficult and commands hard to parse.

**Prevention:** Keep dispatch task strings simple and clean. Pass content as plain text arguments without inline Python or complex shell constructs. Reserve complexity for the dispatch script itself, not the invocation.

## [2026-03-11] Parallel Dispatch Plan Ordering
**Error Pattern:** When dispatching Waves 0 and 1 in parallel, both returned `needs_plan_review` and the agent noted "the plans are swapped between the two dispatches"—Wave 0's plan contained Wave 1's content and vice versa.

**Recovery:** Recognized this as "a parallelism artifact, not a quality issue" and re-dispatched all three waves with explicit `--auto-approve-plan` flag and clear wave labels in task descriptions.

**Root Cause:** Parallel dispatch of similar tasks with subtle differences made plan-to-task association ambiguous; the orchestration system couldn't reliably track which plan belonged to which wave when multiple waves ran simultaneously.

**Prevention:** When orchestrating parallel work with potential ordering ambiguity, include explicit phase/wave labels in task descriptions. If plan tracking is critical, consider sequential dispatch of dependent phases, or ensure dispatch system includes wave ID in all returned artifacts.
