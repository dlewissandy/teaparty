## [2026-03-10] Multi-Team Dispatch Output Handoff Protocol

**Context:** Two parallel research tasks were dispatched to produce `citations.md` and `design-evidence.md` for a downstream writing team. After re-dispatch with `--auto-approve-plan`, extensive effort was spent searching multiple directory structures (.sessions/, .worktrees/, case variants POC/poc) to locate output files and confirm task completion. No explicit handoff mechanism existed between research completion and writing team readiness.

**Learning:** Dispatch pipelines with sequential team dependencies require explicit output location agreements and completion signaling. Implicit assumptions about where artifacts land and how consuming teams discover completion create discovery friction that blocks downstream work.

**Action:** For multi-team dispatch chains: (1) specify exact output file path in dispatch task definition (e.g., `projects/POC/.sessions/{session-id}/research/citations.md`); (2) establish completion signal (file exists at agreed path, explicit message to consuming team when ready, or status registry); (3) document the handoff contract upfront so consuming team knows where to look and when to poll. Consider whether the producing team should directly notify downstream teams via SendMessage when outputs are ready, rather than requiring polling.
