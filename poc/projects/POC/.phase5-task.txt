Implement Phase 5 of the Hierarchical Agent Memory System: reinforcement tracking.

## Prerequisites

Phases 1–4 must be complete. The system now has:
- YAML frontmatter with reinforcement_count and last_accessed fields on every entry
- compact_memory.py that uses reinforcement_count in prominence scoring
- memory_indexer.py that indexes reinforcement_count for retrieval scoring

## What to build

### 1. Create poc/projects/POC/scripts/track_reinforcement.py

This script runs at end-of-session and updates reinforcement_count + last_accessed in memory
entries that were referenced during the session.

Strategy: An entry is "reinforced" when a chunk from it was retrieved and served to the agent
as context. We detect this by looking at which chunk IDs were in the retrieval results and
cross-referencing against the entries in the memory files.

Simpler heuristic (use this — avoid over-engineering):
For each memory file, look for content overlap between the execution stream and memory entries.
An entry is reinforced if its "Learning:" or "Signal:" or key phrase appears verbatim (or nearly)
in the execution stream. Use a line-by-line text match: if any 6+ consecutive words from the
entry body appear in the exec stream, count it as reinforced.

Alternatively (and preferably): maintain a JSON log of which chunk IDs were retrieved by
memory_indexer.py at session start. Use this to update reinforcement.

For this implementation, use the simpler overlap approach:

Usage:
  python3 track_reinforcement.py \
      --stream <.exec-stream.jsonl> \
      --memory-files <file1> [<file2> ...] \
      [--window 8]

Algorithm:
1. Extract all text from the execution stream (assistant text blocks and human turns).
2. For each memory file, parse into entries using the same parse logic as compact_memory.py.
3. For each entry's body text, extract n-grams of length --window (default 8 words).
4. If any n-gram from the entry body appears in the exec stream text, the entry is reinforced.
5. For reinforced entries: increment reinforcement_count by 1, update last_accessed to today.
6. Rewrite the memory file with updated frontmatter for reinforced entries.

Edge cases:
- If exec stream does not exist or is empty: exit cleanly with no changes.
- If a memory file has only legacy entries (no frontmatter): skip without error.
- If reinforcement_count is not in frontmatter: treat as 0, set to 1 after reinforcement.

The script must update entries in-place: parse the file, find reinforced entries by ID, update
their frontmatter (reinforcement_count and last_accessed), rewrite the file.

Module docstring: describe purpose, algorithm, and usage.

### 2. Edit poc/projects/POC/run.sh to integrate track_reinforcement.py

READ the file first to get exact line numbers.

After the learning extraction section (after the promote_learnings.sh calls, around lines 436-483),
before the "Stop the tail" comment, add:

  # Reinforce memory entries referenced during execution
  MEMORY_FILES_TO_REINFORCE=()
  [[ -f "$POC_PROJECT_DIR/MEMORY.md" ]]     && MEMORY_FILES_TO_REINFORCE+=("$POC_PROJECT_DIR/MEMORY.md")
  [[ -f "$POC_OUTPUT_DIR/MEMORY.md" ]]      && MEMORY_FILES_TO_REINFORCE+=("$POC_OUTPUT_DIR/MEMORY.md")
  [[ -f "$POC_PROJECT_DIR/OBSERVATIONS.md" ]] && MEMORY_FILES_TO_REINFORCE+=("$POC_PROJECT_DIR/OBSERVATIONS.md")
  [[ -f "$POC_PROJECT_DIR/ESCALATION.md" ]] && MEMORY_FILES_TO_REINFORCE+=("$POC_PROJECT_DIR/ESCALATION.md")

  if [[ ${#MEMORY_FILES_TO_REINFORCE[@]} -gt 0 && -f "$INFRA_DIR/.exec-stream.jsonl" ]]; then
    python3 "$SCRIPT_DIR/scripts/track_reinforcement.py" \
      --stream "$INFRA_DIR/.exec-stream.jsonl" \
      --memory-files "${MEMORY_FILES_TO_REINFORCE[@]}" 2>/dev/null || true
  fi

### 3. Write a test: poc/projects/POC/scripts/tests/test_phase5_reinforcement.py

Tests must run with: python3 -m pytest poc/projects/POC/scripts/tests/test_phase5_reinforcement.py -v

test_reinforcement_count_increments:
  Create a memory file with a single entry (importance=7, reinforcement_count=0).
  Create a mock exec stream that contains a verbatim phrase from the entry body.
  Run track_reinforcement.py on the file.
  Read the updated file. Verify reinforcement_count is now 1.

test_unreferenced_entry_unchanged:
  Create a memory file with an entry whose content does NOT appear in the exec stream.
  Run track_reinforcement.py.
  Verify reinforcement_count is still 0.

test_last_accessed_updated:
  Create an entry with last_accessed set to a past date.
  Run track_reinforcement.py with a stream that references it.
  Verify last_accessed is updated to today.

test_legacy_entry_skipped_safely:
  Create a memory file with a legacy entry (## [date] format, no frontmatter).
  Run track_reinforcement.py. Verify no crash and file is not corrupted.

test_empty_stream_no_changes:
  Create a memory file with reinforcement_count=0.
  Run track_reinforcement.py with an empty stream file.
  Verify file is unchanged.

test_multiple_reinforcements_cumulative:
  Create an entry with reinforcement_count=3.
  Run track_reinforcement.py with a stream that references it.
  Verify reinforcement_count is now 4.
