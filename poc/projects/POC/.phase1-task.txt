Implement Phase 1 of the Hierarchical Agent Memory System: entry-level structured metadata and compaction.

## Context

The POC memory system lives at poc/projects/POC/scripts/. The relevant files are:
- scripts/summarize_session.py: extracts learnings from session streams; appends plain-text entries
- scripts/promote_learnings.sh: hierarchical rollup (dispatch to team to session to project to global); pure append
- scripts/memory_indexer.py: SQLite FTS5 retrieval pipeline (read only in this phase)

The current memory entry format is plain markdown:
  ## [2026-03-01] Session Learning
  **Context:** ...
  **Learning:** ...
  **Action:** ...

The problem: there is no entry-level metadata, no compaction, no retirement of stale entries.

## What to build

### 2. Create poc/projects/POC/scripts/compact_memory.py

This is the compaction engine. It reads a memory file, scores entries by prominence, retires low-prominence entries, resolves contradictions, and REWRITES the file in place (does NOT append).

Prominence formula:
  prominence = importance * recency_decay * reinforcement_multiplier
  where:
    HALF_LIFE_DAYS = 30.0
    lambda_d = math.log(2) / HALF_LIFE_DAYS
    recency_decay = math.exp(-lambda_d * age_days)     # age_days from created date
    reinforcement_multiplier = 1.0 + math.log(1.0 + reinforcement_count)
    RETIRE_THRESHOLD = 0.3 (default, overridable via CLI)

Legacy entries (no frontmatter): assign defaults importance=5, reinforcement_count=0, phase=operational, use today as created date (they are treated as recent and not auto-retired).

Entry parsing state machine:
  State: BETWEEN_ENTRIES, IN_FRONTMATTER, IN_BODY
  - BETWEEN_ENTRIES: scan for "---" line (opens frontmatter) or "## [" line (legacy entry start)
  - IN_FRONTMATTER: collect lines until closing "---", then switch to IN_BODY
  - IN_BODY: collect lines until next "---" line that would open new frontmatter OR "## [" line

  A "---" on its own line is ambiguous: after we see two "---" lines (open and close of frontmatter), the NEXT "---" starts a new entry. Track a frontmatter_count to distinguish.

Phase compatibility (--current-phase):
  - current_phase=implementation: retire entries where phase=specification
  - current_phase=testing: retire entries where phase=specification OR phase=planning
  - other phases: no cross-phase retirement

Arguments:
  --file FILE           Path to the memory markdown file to compact (modified in place)
  --current-phase PHASE Current task phase for cross-phase retirement (optional)
  --threshold FLOAT     Prominence threshold (default 0.3)
  --dry-run             Print what would happen without writing

Output: rewrite the file with only retained entries. Print to stderr: kept N, retired M entries.

Docstring: Include a module-level docstring explaining the purpose, prominence formula, and usage.

### 3. Update promote_learnings.sh

After EACH call to summarize_session.py that writes to a MEMORY.md file, add a compact_memory.py call on the target file.

In the "team" case: after the summarize_session.py call for each team, add:
  python3 "$SCRIPT_DIR/compact_memory.py" --file "$SESSION_DIR/$team_name/MEMORY.md" || true

In the "session" case: after the summarize call, add:
  python3 "$SCRIPT_DIR/compact_memory.py" --file "$SESSION_DIR/MEMORY.md" || true

In the "project" case: after the summarize call, add:
  python3 "$SCRIPT_DIR/compact_memory.py" \
    --file "$PROJECT_DIR/MEMORY.md" \
    ${POC_TASK_PHASE:+--current-phase "$POC_TASK_PHASE"} || true

In the "global" case: after the summarize call, add:
  python3 "$SCRIPT_DIR/compact_memory.py" --file "$PROJECTS_DIR/MEMORY.md" || true

For prospective, in-flight, corrective scopes: add compact calls on "$PROJECT_DIR/MEMORY.md" after each summarize call.

### 5. Create poc/projects/POC/scripts/tests/test_phase1_memory.py

Write pytest tests verifying Phase 1. Use only stdlib + pytest (no other deps). Run with:
  python3 -m pytest poc/projects/POC/scripts/tests/test_phase1_memory.py -v

Required tests (all must pass):

test_entry_has_yaml_frontmatter:
  Build a string in the new format with frontmatter + body. Parse with PyYAML or a simple parser.
  Verify all required fields present: id, type, importance, phase, status, reinforcement_count, created, last_accessed.

test_compact_retires_low_prominence:
  Create a temp memory file with two entries:
    Entry A: importance=8, created=today, reinforcement_count=0 (high prominence)
    Entry B: importance=2, created=90 days ago, reinforcement_count=0 (low prominence)
  Run compact_memory.main() or subprocess. Verify entry B is not in the output file.

test_compact_no_monotonic_growth:
  Create a temp memory file. Append 3 entries (2 old low-importance, 1 recent high-importance).
  Call compact. Append 2 more old low-importance entries. Call compact again.
  Verify final file has fewer entries than the 5 total appended (compaction removed some).

test_compact_preserves_high_prominence:
  Create an entry with importance=9, created=today. Compact. Verify entry is still present.

test_compact_removes_retired_entries:
  Create an entry with status=retired in frontmatter. Compact. Verify it is removed.

test_phase_transition_retires_spec:
  Create two entries: one with phase=specification (old), one with phase=implementation (recent).
  Run compact with --current-phase implementation.
  Verify spec entry is removed, implementation entry is kept.

test_legacy_entry_handled:
  Create a file with a legacy entry (## [date] format, no frontmatter).
  Compact should not crash and should NOT remove the legacy entry (defaults give it enough prominence).

test_prominence_formula:
  Import the prominence calculation directly from compact_memory.py.
  Verify numerically:
    importance=10, age=0, reinforcement=0: prominence = 10.0 (approximately)
    importance=5, age=30, reinforcement=0: prominence approx 2.5 (half-life=30 days)
    importance=5, age=0, reinforcement=10: prominence = 5 * 1.0 * (1+log(11)) approx 11.99
