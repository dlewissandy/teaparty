# Guides

Practical, task-oriented walkthroughs for using TeaParty. Start with [Quickstart](quickstart.md) if you have never run a session. Come back for the others when you need them.

## For operators

- **[Quickstart](quickstart.md)** — Install TeaParty, open the dashboard, and run your first session. About 10 minutes.
- **[Steering a Session](steering-sessions.md)** — What to do *while* a session is running: chat with the Office Manager, intervene or withdraw, pause and resume, and calibrate the human proxy through direct conversation.
- **[Configuring Teams](configuring-teams.md)** — Add agents, modify workgroup rosters, and create skills through the configuration lead — the right way to change `.teaparty/` without editing YAML by hand.
- **[Project Onboarding](project-onboarding.md)** — Register a new project so the office manager can dispatch work to it.

## If you want to understand what you're doing

- **[Organizational Model](../overview.md)** — the team hierarchy (OM / project lead / workgroup) that these guides operate on.
- **[Architecture](../systems/index.md)** — the six systems that implement it. Each system's `index.md` has a status section telling you what's operational today.
- **[Reference](../reference/folder-structure.md)** — folder structure, team configuration schema, built-in teams, dashboard conventions.
