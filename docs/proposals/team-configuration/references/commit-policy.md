# What Gets Committed

Team configuration YAML files are committed — they are shared team configuration. The following must **never** be committed:

- **Proxy memory databases** (`.proxy-memory.db`, `.proxy-confidence.json`) — these contain a model of a specific human's decision-making patterns. They are private to the decider and local to their machine.
- **Stats files** (`.teaparty/stats/`) — ephemeral, generated by the orchestrator.
- **Session data** (`.teaparty/sessions/`) — ephemeral working state.
- **Context scratch files** (`.context/`) — ephemeral job working memory.

A `.teaparty/.gitignore` and project-level `.gitignore` entries must exclude these paths. This is a privacy requirement — proxy memory contains personally identifiable behavior patterns that must not leave the decider's machine. See [messaging compliance](../../messaging/references/compliance.md).
