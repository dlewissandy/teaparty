Implement Phase 2 of the memory lifecycle architecture: INTENT.md lifecycle isolation.

## Context
Currently INTENT.md is written to the session worktree, then committed via 'git add -A' and squash-merged into the project repo's main branch. On the next session, run.sh loads this prior INTENT.md as persistent context (around line 228), contaminating future sessions with stale task intent.

The fix: after intent approval, copy INTENT.md to the session infra directory (gitignored), then remove it from the session worktree so it is not committed. Update the two downstream consumers to use the archived copy.

- poc/projects/POC/run.sh
