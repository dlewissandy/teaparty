PURPOSE OF PASS 3: Rewrite for external audiences — not just friendlier, but make the research contribution legible. An external reader (peer researcher, headhunter, prospective collaborator) hitting these docs should not encounter walls: undefined terms, bureaucratic phrasing, passive constructions, or research claims without grounding. Pass 3 does NOT restructure (Pass 1) and does NOT change facts (Pass 2). It rewrites prose for clarity, precision, and appropriate register.

RULES:
1. Every document must open with a single clear sentence stating its subject. If a doc starts with background or context instead of stating what it IS, rewrite the opening.
2. Terms defined at first use: CfA -> Conversation for Action (CfA), liaison agent -> define briefly on first use, uber team -> explain what it means when first mentioned, memsearch -> briefly note it is a local hybrid search tool.
3. Language is confident and direct. Replace passive constructions where they obscure the actor. Replace bureaucratic phrasing (it is the case that, in order to, with respect to) with direct equivalents.
4. Research contribution is named as a contribution. Do not just describe behavior — state what the design decision IS and why it matters. E.g., not the system uses a state machine but using an explicit state machine makes approval gates and backtrack transitions formal and auditable.
5. The four-pillar framing from index.md is the anchor. Each of the four pillar docs (hierarchical-teams.md, intent-engineering.md, learning-system.md, human-proxies.md) should open with a sentence that connects it to the pillar. They do not need to repeat the full pillar text — one sentence of orientation is enough.
6. Context rot must be grounded on first use. When the term appears, explain it briefly: context window degradation over long multi-step tasks, where earlier instructions lose salience and the model begins optimizing for recent context. This is the problem that hierarchical teams and context scoping address.
7. The dogfooding story, if mentioned, should be framed as interesting — the POC builds itself using the POC's own team structure. One sentence is enough; do not belabor it.
