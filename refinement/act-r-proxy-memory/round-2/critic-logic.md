# Logical Review — Round 2

## Contradictions

### 1. Confidence as measured signal vs. LLM output
Draft-1 introduces confidence as a first-class signal for triggering surprise extraction (sensorium, lines 82-83: "Confidence delta exceeds threshold... |posterior_confidence - prior_confidence| > 0.3"). But the document never specifies how confidence is extracted from LLM outputs. LLMs do not natively produce calibrated confidence scores. The LLM's prose may contain language like "I'm 85% confident," but parsing and interpretation of such language is lossy and inconsistent across prompts. Yet the alarm logic (moderate surprise threshold at 0.3) treats confidence as if it were a precise measurement. Either confidence is derived from a separate forward pass (adding LLM calls to the cost model) or it is inferred from prose (introducing implicit parsing logic not documented). The document cannot have both a precise threshold mechanism and an undefined measurement method.

### 2. "Surprises are rare" vs. "surprises build the attention model"
Sensorium, lines 72-73: "The prior-to-posterior shift tells us which artifact features drove the prediction change... Over time, accumulated deltas build a map of what this human attends to." Lines 155-163: "learned attention is built from surprising interactions (a minority of gates)... a human who reviews 100 plans and is surprised by 15 of them has learned attention from those 15 cases, not from the 85 routine approvals." If surprises occur in only 15% of interactions, the attention model is built from sparse, noisy signal. Yet line 165 claims: "The next time the proxy reaches PLAN_ASSERT, its Pass 1 prior already reflects this learned attention." For a learned attention model to be reliable enough to feed the prior, the prior itself must be accurate — creating a circular dependency. The 85% routine approvals reinforce the prior (lines 163, 72) but do not feed into the attention model. Over time, the routine cases and the surprising cases diverge in their contribution to different systems. This is not inherently contradictory, but the document's claim that "learned attention accumulates gradually" to the point where "the proxy has earned the right to act autonomously" (lines 165-167) depends on an unstated assumption about the convergence rate and signal quality of a 15% sample.

### 3. EMA separation vs. EMA influence on memory
Act-r-proxy-memory.md, lines 25-26: "The new design separates them: ACT-R memory drives the proxy's behavior... EMA observes the outcomes and reports on system health." This states EMA is decoupled from decision-making. Yet act-r-proxy-mapping.md, lines 179: "A declining EMA signals that upstream agents are producing worse artifacts, not that the proxy should change its behavior. Note that EMA trend information does flow into the proxy's decisions indirectly: if upstream quality degrades, the proxy encounters more corrections and surprises, producing correction chunks with richer salience deltas." This claims EMA trend information flows into decisions indirectly through the memory system — the mechanism is that degradation produces more surprise chunks. But this is not separation. EMA generates a degradation trend; the proxy responds with different retrieval behavior because correction chunks now have higher activation/frequency. The causal path is EMA degradation → more corrections → richer chunks → different memory state → different behavior. This is functionally equivalent to EMA influencing behavior, just with the memory system as a mediating mechanism. The document asserts separation while describing integration.

## Non Sequiturs

### 1. Upstream context "enters memory through two existing dimensions" without overlap
Sensorium, lines 25: "Upstream context is sensed by the proxy and used in Pass 2... It enters memory through two existing embedding dimensions: the artifact embedding... and the stimulus embedding." Immediately preceding (lines 24-25): "A dedicated upstream embedding dimension was considered but removed because it overlapped with artifact and stimulus without adding discriminating power." The logic is: upstream context overlaps with artifact and stimulus enough to remove a dedicated dimension, yet also distinctly enters both artifact and stimulus dimensions. If the overlap is sufficient to eliminate a dedicated dimension, how does upstream context maintain its identity when distributed across two dimensions that already capture the artifact and the question? Retrieval scores chunks by cosine similarity across dimensions; a chunk with high artifact-similarity due to upstream context cannot be distinguished from a chunk with high artifact-similarity due to artifact content alone. The document removes the dimension, then asserts the content still enters the system. One of these must be wrong.

### 2. "Two-pass auto-approval demonstrates understanding" from prior-posterior agreement
Sensorium, lines 166-167: "the proxy has earned the right to act autonomously... because it has demonstrated that it attends to what the human would attend to, inspects the artifact through two-pass prediction, and finds nothing unexpected." The evidence cited is "finds nothing unexpected" — meaning the prior and posterior agree. But prior-posterior agreement is exactly what Round 1 criticized as conflating prediction accuracy with understanding. If the proxy's prior predicts "approve" and the posterior also says "approve," the prior was accurate. But accuracy does not imply the proxy examined the artifact. The proxy could have a highly accurate prior because it is pattern-matching on (state, task_type) without ever attending to artifact-specific content. Two-pass execution (running both passes) is not the same as two-pass inspection (examining the artifact in Pass 2). The document claims the posterior "examined the artifact and found it matches expectations." But "matches expectations" is indistinguishable from "matches a shallow pattern in the prior." The sensorium document's own defense (lines 200-202) asserts the proxy "examined the artifact" and "got the right answer because I was looking at the right things." Yet the autonomy criterion is posterior agreement, which is silent on what the proxy was looking at.

### 3. Cold start produces "richer interaction data" therefore faster convergence
Act-r-proxy-mapping.md, lines 169: "the new system generates richer interaction data per gate (questions, responses, reasoning, structured chunks with embeddings) compared to a single binary outcome update... whether the proxy reaches useful retrieval quality in fewer interactions depends on whether the additional data dimensions actually improve retrieval, which is the very thing the ablations are designed to test." This is stated as an open question — the document acknowledges it's unknown. Yet act-r-proxy-memory.md, lines 40-41 (the autonomy note) asserts: "the new design does permit autonomous proxy action... when the proxy has demonstrably inspected the artifact via two-pass prediction and its prior-posterior agreement reflects genuine understanding." The autonomy mechanism is built into the design at act-r-proxy-memory.md even though act-r-proxy-mapping.md flags the empirical dependency (whether richer data helps) as unresolved. The system is designed assuming an affirmative answer to an explicitly open question.

## Unstated Assumptions

### 1. Confidence values from the LLM are measurable and stable
The surprise extraction threshold (sensorium, lines 82-83) uses |posterior_confidence - prior_confidence| > 0.3. This assumes: (a) the LLM produces a confidence value on each pass, (b) the value is numeric and comparable across passes, (c) a 0.3 threshold is meaningful in the LLM's output distribution, and (d) this threshold is stable across different artifacts and contexts. The document states (line 83) that "LLM-generated confidence scores are poorly calibrated, so this threshold needs empirical calibration during shadow mode." This acknowledgment undercuts the assumption: if confidence is poorly calibrated, why use it for an alarm threshold at all? The system depends on confidence thresholds for a critical decision (when to extract salient percepts, when to activate surprise learning), but confidence itself is unstable. This is not necessarily wrong — the document flags it for empirical validation — but it is an unstated and risky dependency.

### 2. Chunks matching on only one embedding dimension provide useful retrieval signal
Act-r-proxy-mapping.md, lines 140-141: "a chunk with high similarity on 2 populated dimensions out of 5 gets (sim1 + sim2 + 0 + 0 + 0) / 5... while a chunk with moderate similarity across all 5 gets (sim1 + sim2 + sim3 + sim4 + sim5) / 5. This rewards breadth of matching: chunks that match narrowly on fewer dimensions score higher than chunks matching narrowly on fewer dimensions." Wait — the statement contradicts itself. Rereading: the design divides by 5 always, not by the number of populated dimensions. This penalizes sparse matching (high on situation, zero on others) and rewards dense matching. But the document assumes that a chunk matching on situation alone (e.g., "what happens at PLAN_ASSERT?") provides actionable retrieval signal. Yet the dense-matching penalty means such chunks are scored (sim_situation + 0 + 0 + 0 + 0) / 5 = 0.2 * sim_situation. A chunk matching moderately on all five dimensions is scored at 0.2 * (5 moderate sims). For sparse matching to win, it must have sim_situation > 5 * moderate_sim, which is unlikely. The system design assumes sparse matching is useful but penalizes it mathematically. This is not inherently wrong — sparse matching may not be useful — but the assumption is unstated.

### 3. "The dialog happened inside the proxy's reasoning" is meaningful validation
Sensorium, line 167: "The dialog happened inside the proxy's reasoning. The proxy earned its autonomy by demonstrating understanding, not by accumulating a high scalar." This reframes internal LLM reasoning as equivalent to human-proxy dialog. The assumption is that internal reasoning (the proxy's two-pass chains of thought) is sufficient validation for autonomous action. But internal reasoning is opaque — the proxy's LLM chain of thought cannot be inspected by the human. The sensorium document's own definition of learning (lines 155-163) is that human feedback teaches the proxy by enabling it to build accuracy on (state, task_type) pairs. If the proxy learns from human corrections, it learns from *dialog* with the human, not from internal reasoning. Internal reasoning is how the proxy applies what it learned; it is not the source of learning. Autonomous action (no human escalation) skips the very loop (human feedback) that trains the system. The document assumes internal reasoning is sufficient validation for skipping the training loop. This is possible if the proxy's prior is accurate enough that the posterior is reliable without correction. But that requires the prior to be *already well-trained*, which requires previous human dialog. The assumption that internal reasoning replaces dialog only holds after extensive human feedback, and the document does not bound this learning period.

## Assessment

The document set is structurally more coherent than Round 1's draft — the REINFORCE removal is now explicit, and the EMA role is more carefully bounded. However, the logical weaknesses have shifted rather than resolved:

1. **Precision threshold on undefined measurement.** The surprise extraction mechanism depends on confidence as a precise threshold, but confidence is explicitly stated as poorly calibrated. The system is built on a foundation the document admits is unstable.

2. **Sparse signal training a reliable prior.** The attention model is learned from 15% of interactions (surprises). The document assumes this sparse signal is sufficient to train a prior that is accurate enough to skip human dialog. The learning curve is not characterized.

3. **EMA reintegration through indirection.** The document claims to separate ACT-R memory from EMA monitoring, but then describes EMA trends flowing into behavior through the memory system. This is integration, not separation, just with the memory system as intermediary.

4. **Two-pass execution conflated with two-pass inspection.** The autonomy mechanism depends on showing that the proxy examined the artifact (two-pass inspection) but measures only whether the two passes agree (two-pass execution). These are not equivalent. The system can satisfy the formal criterion without meeting the intent.

The document is internally consistent within each section, but the sections make assumptions about how their components interact that are not reconciled. The migration path (Phase 1 shadow mode, Phase 2 dialog mode, Phase 3 integrated) is sound, but the criteria for transition are underspecified. At what point does sparse surprise signal become reliable enough to support autonomy? The document flags this for empirical validation in shadow mode (good) but then asserts autonomy is permissible before shadow mode ends (risky).

The design is defensible but depends on empirical facts that the document does not establish: that confidence thresholds are calibrable, that sparse surprise signal trains reliable priors, and that agreement between passes is evidence of actual inspection. These are open questions, not settled design facts.
